Bug: Sidebar Expand/Collapse Icons Display Incorrectly

The MacOSSidebar incorrectly uses the flatIndex when checking the _expandedIndices set in _buildAwesomeSidebarItem and _buildSidebarItem. Since _expandedIndices stores originalIndex, this mismatch causes the expand/collapse icons to display the wrong state and prevents expandable items from toggling correctly.
