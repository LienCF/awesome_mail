Bug: Email Rollback Fails, UI/Backend Mismatch

The rollback logic for _onEmailArchive and _onEmailDelete attempts to re-insert an email using an emailIndex from the original list. After an optimistic removal, this index is incorrect for the modified list, potentially misplacing the email or causing an index error. Additionally, _onEmailArchive performs an optimistic UI update but lacks a backend implementation, leading to an inconsistent state where emails are removed from the UI but not archived on the server.
